用户与用户组

Linux一般将文件可存取访问的身份分为三个类别，分别是owner、group、others，且这三个身份各有read、write、execute等权限。

文件所有者：由于Linux是个多用户、多任务的系统，因此可能常常会有很多人同时使用这台主机来进行工作的情况发生，为了考虑每一个人的
隐私权以及每个人喜好的工作环境，因此这个“文件所有者”的角色就显得相当重要了。

用户组：在Linux系统下，可以进行简单的文件权限设置，就能限制非自己团队（用户组）的其他人不能够阅读内容，而且也可以让自己的团队
成员可以修改所创建的文件。同时，如果个人还有隐私文件，仍然可以设置成让自己的团队成员看不见。
注：每个账号都可以拥有多个用户组的支持。

其他人：用户组之外的访问者；

注：root可以任意访问任何文件

Linux用户身份与用户组记录的文件
在Linux系统下，默认的情况是所有的系统上的账号与一般身份用户，还有root的相关信息，都是记录在/etc/password 这个文件内；
个人的密码则是记录在/etc/shadow 这个文件下；linux所有的组名都记录在/etc/group 这个文件下。

Linux文件权限概念
命令： ls -al
ls是list的意思，重点在显示文件的文件名与相关属性。al则表示列出所有的文件详细的权限和属性（包含隐藏文件）。

格式：[ - ] [ rw- ] [ r-- ] [ r-- ] 1 root root 42304 Sep 4 18:26 install.log

第一列代表文件的类型与权限（permission）
	第一个字符代表这个文件是“目录、文件或链接文件”
	[ d ] ：目录
	[ - ] ：文件
 	[ l ] ：连接文件
	[ b ] ：设备文件里的可供存储的接口设备
	[ c ] ：设备文件里的串行端口设备，例如键盘、鼠标
[ r ]代表可读（read），[ w ]代表可写（write），[ x ]代表可执行（execute）。

第一组为“文件所有者的权限”；第二组为“同用户组的权限”；第三组为“其他人的权限”。

第二列数字代表有多少文件连接在此节点（i-node）；

第三列表示这个文件（或目录）的“所有者账号”；

第四列表示这个文件的所属用户组；

第五列表示这个文件的容量大小，默认单位是“B”；

第六列表示这个文件的创建日期或者是最近的修改日期；

第七列表示这个文件的文件名。
